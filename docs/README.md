# Documentation Hub

Tài liệu trong thư mục này được chia thành 2 nhóm:

- `CURRENT_STATE.md`: tài liệu trạng thái triển khai thực tế của dự án ở thời điểm hiện tại
- các file `*_SPEC.md`, `*_PLAN.md`, `*_STRATEGY.md`: tài liệu theo chủ đề để đào sâu từng phần

Nếu bạn mới vào dự án, nên đọc theo thứ tự:

1. [`CURRENT_STATE.md`](./CURRENT_STATE.md)
2. [`API_SPEC.md`](./API_SPEC.md)
3. [`DB_SCHEMA.md`](./DB_SCHEMA.md)
4. [`UX_FLOWS.md`](./UX_FLOWS.md)
5. [`TEST_PLAN.md`](./TEST_PLAN.md)
6. [`RELEASE.md`](./RELEASE.md)

## Danh mục

- [`CURRENT_STATE.md`](./CURRENT_STATE.md): tổng quan sản phẩm, kiến trúc, setup local, trạng thái tính năng và known gaps
- [`PRODUCT_SPEC.md`](./PRODUCT_SPEC.md): mục tiêu sản phẩm, personas, phạm vi MVP
- [`TECH_SPEC.md`](./TECH_SPEC.md): stack kỹ thuật, kiến trúc frontend/backend, risk scoring
- [`API_SPEC.md`](./API_SPEC.md): mô tả endpoint và response shape hiện tại
- [`DB_SCHEMA.md`](./DB_SCHEMA.md): mô hình dữ liệu thực tế theo Prisma schema
- [`UX_FLOWS.md`](./UX_FLOWS.md): luồng trải nghiệm hiện có cho employee, manager, admin
- [`TEST_PLAN.md`](./TEST_PLAN.md): chiến lược test và phạm vi test hiện tại
- [`RELEASE.md`](./RELEASE.md): quy trình release, tag, deploy, verify
- [`SCALING_STRATEGY.md`](./SCALING_STRATEGY.md): định hướng mở rộng quy mô
- [`AI-SLOP-CHECKLIST.md`](./AI-SLOP-CHECKLIST.md): checklist tránh AI slop
- [`AI-SLOP-AUDIT.md`](./AI-SLOP-AUDIT.md): audit ghi nhận chất lượng
- [`openapi.yaml`](./openapi.yaml): placeholder OpenAPI file trong repo
